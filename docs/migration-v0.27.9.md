# v0.27.9 迁移说明

本次版本不引入新的配置项，主要增强两类内部行为：

- `intent_analysis` 改为“AI 证据优先 + possible-AI 候选”语义，减少把其它对象误判成当前模型自身。
- SKILL 支持结构化内部文本/多模态结果通道，框架会隐藏技能元信息，只把可用结论送入最终回复。

## 变更摘要

- 名字或别称中带有 `AI`、`bot`、`助手`、`Claude`、`Qwen` 等明显线索的对象，会在意图提示词里被列为“名称上带明确AI线索的对象”。
- 没有这类线索的对象，不再先行硬标记为“人类参与者”，而会作为“需结合上下文判断的对象”交给模型结合最近交互链、aliases 和环境线索判别。
- `self_ai -> other_ai` 的保护性降级现在只在命中其他 AI 或明确 AI 线索对象时触发，不再因为普通对象名字就强行改写。
- SKILL 可返回 `text_blocks`、`multimodal_blocks`、`internal_metadata`；engine 会把文本块和可识别图片隐藏注入下一轮生成。
- 系统提示词与内部技能通道都会明确约束模型不要输出 `internal_metadata`、`mime_type`、`label`、路径、URL 或 JSON 字段名。
- 高热群聊中的 ambient reply 惩罚略微增强，减少多人密集聊天时的抢答。

## 对外影响

- 如果你之前依赖意图提示词里的“群内人类参与者”“当前消息命中的人类名字”这类硬标签做调试观察，升级后会看到更中性的“其它已知对象”“用户侧发言者”“名称含 AI 线索对象”“possible-AI 候选对象”等表述。
- 无需修改现有 `SessionConfig` 或 `OrchestrationPolicy` 配置。
- 若你在编写自定义 SKILL，旧式返回普通 `dict` 仍然完全可用；只有需要更细的内部文本或图片通道时，才需要新增结构化字段。
- `SKILL_COMPLETED` 的外部事件语义保持不变，依旧只表达执行状态，不直接暴露技能结果正文。

## 推荐升级检查

升级后，建议至少覆盖以下场景：

1. 多 AI 群聊里，对象名字带有 `bot` / `AI` 等线索时，确认当前模型不会误把消息当成在叫自己。
2. 普通名字对象（如“小王”）在没有 AI 线索时，确认意图提示词使用中性对象表述，而不是提前贴“人类”标签。
3. 自定义 SKILL 若返回 `text_blocks` 或 `multimodal_blocks`，确认模型能消费结果结论，但最终 assistant 回复不会复述 `internal_metadata`、`mime_type`、`label`、路径或 URL。
4. 在高热群聊里发送非定向普通消息，确认 `reply_mode=auto` 下当前模型比之前更克制；被明确点名时仍能正常回复。
